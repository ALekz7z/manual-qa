## 1. Введение

### 1.1 Цель документа
Этот тест-план описывает стратегию ручного тестирования веб-сайта DemoQA, включая тестирование форм, кнопок, алертов и интерактивных элементов.

### 1.2 Описание проекта
DemoQA — учебный веб-сайт с коллекцией изолированных модулей для тестирования различных элементов интерфейса и функциональности.

**URL:** https://demoqa.com/  
**Тип приложения:** Веб-сайт с демонстрационными элементами

---

## 2. Область тестирования

### Что тестируем:
- **Forms → Practice Form**: Валидация полей (имя, email, телефон), загрузка файла, работа с дата-пикером
- **Elements → Buttons**: Реакция на клик, двойной клик, правый клик
- **Alerts → Browser Alerts**: Появление алертов, текст сообщений, кнопки OK/Cancel
- **Interactions → Draggable**: Перетаскивание элементов, сохранение позиции
- Базовый UI/UX всех модулей

### Что НЕ тестируем:
- Нагрузочное тестирование
- Безопасность и penetration testing
- Мобильные приложения
- API тестирование
- Автоматизированные тесты
- Кроссбраузерное тестирование
