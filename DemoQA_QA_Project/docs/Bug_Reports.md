## Баг-репорты

### **BUG-001: Отсутствие сообщения об ошибке при вводе некорректного Email**

**Статус:** Новый  
**Приоритет:** Medium  
**Серьёзность:** Minor  
**Тип:** Functional/Validation/UI  

**Окружение:** 
- Браузер: Mozilla Firefox
- ОС: Windows 11
- URL: https://demoqa.com/practice-form

**Описание:**  

При вводе некорректного формата email (без символа @) поле ввода подсвечивается красной рамкой, однако текстовое сообщение об ошибке не отображается, а иконка информации (ℹ️) рядом с полем неактивна.



**Шаги воспроизведения:**
1. Перейти на страницу https://demoqa.com/practice-form
2. Заполнить поле "First Name": `John`
3. Заполнить поле "Last Name": `Doe`
4. Ввести в поле "Email": `testexample.com` (без символа @)
5. Кликнуть на кнопку Submit и попытаться отправить форму

**Ожидаемый результат:**  
- Поле Email подсвечивается красной рамкой
- Появляется текстовое сообщение: "Please enter a valid email address" или "Email must contain @ symbol"
- Иконка ℹ️ активна и показывает подсказку при наведении курсора

**Фактический результат:**  
- Рамка поля становится красной 
- Сообщение об ошибке отсутствует 
- Иконка ℹ️ неактивна (не реагирует на наведение и клик) 
- Пользователь не получает информацию о причине ошибки

**Скриншот:**  
screenshots/BUG-001_email_no_error_message.png  

**Комментарии:**  
Баг воспроизводится стабильно. Проблема влияет на UX: пользователи могут не понять причину отказа формы. Рекомендуется добавить текстовое сообщение об ошибке и активировать иконку с подсказкой.

### **BUG-002: Кнопка правого клика не сбрасывает предыдущее сообщение**

**Статус:** Новый  
**Приоритет:** Low  
**Серьёзность:** Trivial  
**Тип:** UI/UX  
**Окружение:** 
- Браузер: Mozilla Firefox 
- ОС: Windows 11
- URL: https://demoqa.com/buttons

**Описание:**  
При последовательном нажатии кнопок (сначала двойной клик, затем правый клик), текст сообщения о правом клике добавляется к предыдущему сообщению, а не заменяет его.

**Шаги воспроизведения:**
1. Перейти на страницу https://demoqa.com/buttons
2. Выполнить двойной клик на кнопке "double click me"
3. Запомнить появившееся сообщение
4. Выполнить правый клик на кнопке "right click me"
5. Проверить отображаемое сообщение

**Ожидаемый результат:**  
Отображается только сообщение о последнем действии: "You have done a right click"

**Фактический результат:**  
Сообщения накапливаются: "You have done a double click" + "You have done a right click"

**Скриншот:**  
screenshots/BUG-002_buttons_message_accumulate.png

**Комментарии:**  
Не критично для функциональности, но влияет на читаемость результатов тестирования. Рекомендуется очищать предыдущее сообщение перед отображением нового.

### **BUG-003: Модальное окно алерта перекрывает кнопку закрытия на мобильных устройствах**

**Статус:** Новый  
**Приоритет:** Medium  
**Серьёзность:** Minor  
**Тип:** UI/Responsive  
**Окружение:** 
- Браузер: Google Chrome (режим эмуляции мобильного устройства)
- ОС: Windows 10/11
- URL: https://demoqa.com/alerts

**Описание:**  
При открытии модального окна (Modal) на узком экране (ширина менее 768px), кнопка закрытия (×) частично перекрывается содержимым модального окна или выходит за пределы видимой области.

**Шаги воспроизведения:**
1. Перейти на страницу https://demoqa.com/alerts
2. Открыть инструменты разработчика (F12)
3. Включить режим эмуляции мобильного устройства (iPhone SE или аналогичный)
4. Нажать кнопку "Modal"
5. Попытаться найти и нажать кнопку закрытия (×)

**Ожидаемый результат:**  
Кнопка закрытия полностью видна и доступна для нажатия на всех размерах экрана

**Фактический результат:**  
Кнопка закрытия частично скрыта или труднодоступна для нажатия на мобильных устройствах

**Скриншот:**  
screenshots/BUG-003_modal_mobile_overlay.png

**Комментарии:**  
Воспроизводится на экранах шириной менее 768px. Влияет на доступность интерфейса для мобильных пользователей.
