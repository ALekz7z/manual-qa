## Баг-репорты

### **BUG-001: Сообщение об ошибке не отображается при пустом поле First Name**

**Статус:** Новый  
**Приоритет:** Medium  
**Серьёзность:** Minor  
**Тип:** Functional  
**Окружение:** 
- Браузер: mozilla 148.0.2
- ОС: Windows 11
- URL: https://www.saucedemo.com/checkout-step-one.html

**Описание:**  
При оформлении заказа, если оставить поле "First Name" пустым и нажать кнопку "Continue", ошибка не отображается красным текстом под полем, хотя форма не отправляется.

**Шаги воспроизведения:**
1. Авторизоваться как `standard_user` / `secret_sauce`
2. Добавить любой товар в корзину
3. Нажать на иконку корзины
4. Нажать кнопку "Checkout"
5. Оставить поле "First Name" пустым
6. Заполнить "Last Name": `Doe` и "Zip": `12345`
7. Нажать кнопку "Continue"

**Ожидаемый результат:**  
Под полем "First Name" появляется сообщение об ошибке красного цвета: "Error: First Name is required"

**Фактический результат:**  
Форма не отправляется, но сообщение об ошибке не отображается визуально

**Скриншот:**  
screenshots/BUG-001_empty_firstname

**Комментарии:**  
Баг воспроизводится стабильно.

### **BUG-002: Кнопка "Remove" не меняет цвет при наведении**

**Статус:** Новый 
**Приоритет:** Low  
**Серьёзность:** Trivial  
**Тип:** UI/UX  
**Окружение:** 
- Браузер: mozilla 148.0.2
- ОС: Windows 11
- URL: https://www.saucedemo.com/cart.html

**Описание:**  
Кнопка "Remove" в корзине не меняет цвет или не имеет hover-эффекта при наведении курсора, что ухудшает пользовательский опыт.

**Шаги воспроизведения:**
1. Авторизоваться как `standard_user` / `secret_sauce`
2. Добавить товар "Sauce Labs Backpack" в корзину
3. Перейти в корзину (нажать на иконку корзины)
4. Навести курсор на кнопку "Remove"

**Ожидаемый результат:**  
Кнопка "Remove" меняет цвет фона или текста при наведении

**Фактический результат:**  
Кнопка не реагирует на наведение курсора, визуальных изменений нет

**Скриншот:**  
screenshots/BUG-002_remove_button_hover.png

**Комментарии:**  
Не критично для функциональности, но влияет на UX.

### **BUG-003: Отсутствует подтверждение перед очисткой корзины**

**Статус:** Новый  
**Приоритет:** Low  
**Серьёзность:** Minor  
**Тип:** UX  
**Окружение:** 
- Браузер: mozilla 148.0.2
- ОС: Windows 11
- URL: https://www.saucedemo.com/cart.html

**Описание:**  
При удалении всех товаров из корзины нет подтверждающего диалога. Пользователь может случайно очистить корзину без возможности восстановления.

**Шаги воспроизведения:**
1. Авторизоваться и добавить 2-3 товара в корзину
2. Перейти в корзину
3. Последовательно нажать "Remove" для каждого товара

**Ожидаемый результат:**  
При последнем удалении появляется подтверждение: "Вы уверены, что хотите очистить корзину?"

**Фактический результат:**  
Корзина очищается мгновенно без подтверждения

**Скриншот:**  
screenshots/BUG-003_empty_cart_no_confirm.png

**Комментарии:**  
Рекомендация по улучшению UX.
